HTML Forms

Forms are the primary way users submit data to a web server. They contain various input elements that capture different types of information. Every form must be wrapped in a <form> tag and typically includes a submit button to send data.

Form Tag Attributes: ACTION (where to send data) | METHOD (GET or POST) | NAME (form identifier)

Text Input & Email

Text inputs are the foundation of form interactions. They accept single-line text, while the email type provides built-in validation for email addresses.

Key Attributes: SIZE (display width) | MAXLENGTH (character limit) | VALUE (default text) | PLACEHOLDER (hint text)
💻 Code Example
<input type="text" name="username" SIZE="35" placeholder="Enter username"> <input type="email" name="email" placeholder="your@email.com">

Radio Buttons

Radio buttons allow users to select ONE option from a group. All radio buttons in a group must share the same NAME attribute, which ensures only one can be selected at a time.

Key Rules: Same NAME = one group | Only ONE can be selected | Use CHECKED for default | Each needs unique VALUE
💻 Code Example
<input type="radio" name="choice" value="option1"> Option 1 <input type="radio" name="choice" value="option2" checked> Option 2 (default)

Checkboxes

Checkboxes allow users to select MULTIPLE options from a list. Unlike radio buttons, each checkbox is independent. Checkboxes in the same group typically share the same NAME.

Key Rules: Multiple can be selected | Use same NAME for grouping | Use CHECKED for defaults | Each needs unique VALUE
💻 Code Example
<input type="checkbox" name="interests" value="sports"> Sports <input type="checkbox" name="interests" value="music" checked> Music (default)

Dropdown Lists (Select)

Dropdown lists (select elements) allow users to choose from a predefined list of options. They're ideal for long lists of choices where radio buttons would take too much space.

Key Attributes: SIZE="1" (dropdown) | SIZE="n" (scrolling list with n visible rows) | MULTIPLE (allow multiple selections)
💻 Code Example
<select name="country"> <option>Select...</option> <option value="usa">USA</option> <option value="uk">UK</option> <option value="canada" selected> Canada (default) </option> </select>

HTML Tables

Tables are used to display data in rows and columns. They consist of <table>, <thead>, <tbody>, <tr> (rows), <th> (header cells), and <td> (data cells). Tables should be used for tabular data, not for page layout.

Table Structure: <table> contains the entire table | <thead> contains header row(s) | <tbody> contains data rows | <tr> defines a row | <th> defines header cell | <td> defines data cell
📊 Table Structure Code
<table> <thead> <tr> <th>Header 1</th> <th>Header 2</th> <th>Header 3</th> </tr> </thead> <tbody> <tr> <td>Data 1</td> <td>Data 2</td> <td>Data 3</td> </tr> </tbody> </table>

HTML Lists

Lists are fundamental HTML elements used to organize and present information in a structured way. HTML supports three types of lists: unordered lists (with bullets), ordered lists (with numbers), and description lists (with terms and definitions).

Three Types of Lists: <ul> (Unordered List) | <ol> (Ordered List) | <dl> (Description List)

Unordered Lists (<ul>)

Unordered lists display items with bullet points. Each item is marked with a <li> tag. Use <ul> when the order of items doesn't matter.

Unordered List Rules: Each item wrapped in <li> | Default bullet style is disc | Can change with CSS (circle, square) | Order doesn't matter
💻 Code Example
<ul> <li>First item</li> <li>Second item</li> <li>Third item</li> </ul>
  • Milk
  • Eggs
  • Bread
  • Butter

Ordered Lists (<ol>)

Ordered lists display items with numbers. Use <ol> when the sequence of items is important, such as steps in a process or a ranking.

Ordered List Rules: Each item wrapped in <li> | Default numbering is 1, 2, 3... | Can use START attribute to begin at different number | TYPE attribute controls style (1, a, A, i, I)
💻 Code Example
<ol> <li>First step</li> <li>Second step</li> <li>Third step</li> </ol> <ol type="a" start="1"> <li>Item A</li> <li>Item B</li> </ol>
  1. Preheat oven to 350°F
  2. Mix dry ingredients
  3. Combine wet ingredients
  4. Bake for 25 minutes

  1. Option A
  2. Option B
  3. Option C

Nested Lists

Lists can be nested inside other lists to create hierarchical structures. This is useful for showing relationships and sub-categories within your data.

Nesting Guidelines: Lists can be nested to any depth | Mix ordered and unordered lists | Each nested list must be inside a <li> tag | Proper indentation improves readability
💻 Code Example
<ul> <li>Fruits <ul> <li>Apples</li> <li>Bananas</li> </ul> </li> <li>Vegetables <ul> <li>Carrots</li> <li>Broccoli</li> </ul> </li> </ul>

Description Lists (<dl>)

Description lists are used to define terms and their descriptions. This is ideal for glossaries, definitions, specifications, and Q&A formats. Uses <dt> (term) and <dd> (definition) tags.

Description List Structure: <dl> contains the entire list | <dt> defines a term | <dd> provides the definition or description | Multiple terms can share one definition or vice versa
💻 Code Example
<dl> <dt>HTML</dt> <dd>HyperText Markup Language</dd> <dt>CSS</dt> <dd>Cascading Style Sheets</dd> <dt>JavaScript</dt> <dd>Programming language for web interactivity</dd> </dl>
HTML
HyperText Markup Language – the standard markup language for creating web pages
CSS
Cascading Style Sheets – used to style and layout web pages
DOM
Document Object Model – a programming interface for HTML documents
